# Release 1.0.0 Roadmap — 2026-04-09

This document outlines the final steps to move `harus-aeron-zig` from its current high-parity state (Phase 10) to a production-ready 1.0.0 release.

## Current Status (Audit 2026-04-09)
The core Aeron implementation in Zig is remarkably complete (~95% parity). Major correctness bugs (CAS ordering, raw_tail pointers, counter memory ordering) have been resolved. The focus for 1.0.0 is on closing the remaining operational gaps and ensuring full educational parity.

### Subsystem Completion
- **Driver Core**: 95%
- **Protocol**: 98%
- **Archive**: 100%
- **Cluster**: 90%
- **Educational Layer**: 85%

---

## 1. Remaining Operational Gaps
These gaps separate the current implementation from full operational parity with `aeron-io/aeron`:

1.  **IPC Advanced Keepalive**: Implementation of `ManyToOneRingBuffer.unblock()` to handle crashed writers that leave negative length sentinels in the ring buffer.
2.  **Cluster Snapshot Coordination**: Implementing actual state serialization of `ClusterConductorState` to an Aeron Archive recording (currently a state-machine stub).
3.  **Cluster Member Discovery**: Support for dynamic member joining and discovery protocols beyond static configuration.
4.  **URI Wildcard Substitution**: Full address resolution for wildcard endpoints and interfaces (e.g., `endpoint=127.0.0.1:*` or `interface=0.0.0.0`).
5.  **Multi-destination-cast (UDP)**: Polishing the UDP MDC control-mode logic for complex deployment scenarios.

---

## 2. Release 1.0.0 Phases

### Phase 11a: Core Fidelity & Hardening
- Implement `ManyToOneRingBuffer.unblock()` for writer recovery.
- Wire Cluster Snapshot logic to the Archive for persistence.
- Implement dynamic member discovery for Cluster.
- Support full URI wildcard substitution in `UdpChannel`.

### Phase 11b: Educational Parity
- Complete all remaining `tutorial/` stubs for Part 5 (Archive) and Part 6 (Cluster).
- Verify all 24 chapters of the tutorial course compile via `make tutorial-check`.
- Sync all `LESSON` annotations in `src/` with their corresponding tutorial stubs.

### Phase 11c: Quality Gate & Performance
- **Interop**: Run the full interop matrix (`make interop`) covering Pub/Sub, Archive, and Cluster.
- **Stability**: Execute 24-hour soak tests (`make stress`) under cluster failover conditions.
- **Performance**: Establish and document the 1.0.0 throughput and latency baseline (`make bench`).

### Phase 11d: Release Execution
- Bump version to `1.0.0` in `build.zig.zon`.
- Update `CHANGELOG.md` with full feature surface.
- Final documentation polish (Architecture, Setup, and Tutorial guides).
